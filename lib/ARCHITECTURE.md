# lib/ — package architecture

Public entry point: `shadcn_ui.dart` (barrel file — every public component and theme is exported here).

## Layout

- `src/app.dart` — `ShadApp` variants (Material/Cupertino/plain) that wire `ShadTheme` into an app.
- `src/components/` — one file per widget, class prefix `Shad` (e.g. `button.dart` → `ShadButton`).
  `form/` holds the form field wrappers.
- `src/theme/`
  - `theme.dart` / `data.dart` — `ShadTheme` inherited widget and `ShadThemeData`
    (`data.g.theme.dart` is generated by `theme_extensions_builder` — regenerate, don't edit).
  - `components/` — one theme class per component; every visual parameter a component accepts
    exists here too, resolved as `widget.x ?? theme.xTheme.x ?? default`.
  - `color_scheme/`, `text_theme/`, `themes/` — shadcn color schemes and default light/dark themes.
- `src/raw_components/` — behavior primitives with no shadcn styling (portal, focusable, etc.), reused by components.
- `src/utils/` — helpers (responsive, positioning, states controller, mouse area, input formatters…).
- `src/i18n/` — slang-generated localizations (edit the slang source, not the output).

## Rules that live here

- No `material.dart` / `cupertino.dart` / barrel-file imports inside `lib/` (CI-enforced; see root `CONSTRAINTS.md`).
- Tests mirror this tree under `test/src/`; a component change without a matching test update is incomplete.
